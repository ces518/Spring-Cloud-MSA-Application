# Microservice Architecture

## 특징
- 기존 개발 방식과 패러다임을 바꿔야함
- 독립적인 서비스로 구성
- 서비스 경계를 잘 구분해야 함
- RESTful API 로 통신 권장
- 외부 시스템을 통해 설정 관리
- Cloud Native 활용
- 동적인 스케일업 / 스케일아웃
- CI/CD
- 시각화

## MSA 도입 전 검토
- 얼마나 변화가 이뤄질 것인가
    - 공수에 대한 고려
- 독립적인 사이클
    - 각 서비스 경계가 잘 구분되어 있는가
- 독립적인 확장성
    - 스케일링이 쉽게 가능한가
- 격리된 오류 환경
    - 해당 서비스 외부로 영향 최소화
- 외부 종속성을 줄여야함
    - 서비스간 종속성 최소화
- 폴리그랏
    - 폴리그랏 패러다임
