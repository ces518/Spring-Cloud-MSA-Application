# 12 Factors
- 클라우드 네이티브 애플리케이션을 구축하기 전에 고려해야할 12가지 항목
- FaaS 를 제공하는 heroku 라는 회사에서 제시함
- https://12factor.net/


1. 코드 통합
2. 종속성 배제
3. 환경설정의 외부관리
4. 백업 서비스의 분리
5. 개발환경과 테스트환경의 분리
6. 상태관리
7. 포트바인딩
8. 동시성
9. 서비스의 올바른 상태유지
10. 개발-운영 환경의 통일
11. 로그의 분리
12. 관리 프로세스

## 12 Factors + 3
- 피보탈에서 3가지 추가 제시
- API first
    - 사용자 측에서 어떻게 사용할것안지를 고려해서 개발해야함 
- Telemetry
    - 모든 지표나 수치는 시각화해서 관리되어야 한다.
- Authentication and authorization
    - 인증 정보는 필수
